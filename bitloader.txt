	I had to modify Lee "improved" Bit Loader Program so it worked
correctly. Not just for my use, but for anybody who want to use it for
whatever. I added a bit time delay between the middle of the last bit of the
byte to the middle of the stop bit, then grab next byte. I added a transfer
portion to this software so it is transferred to the correct location to
auto load the monitor program. I have attached the ASM file for both of them
for your reading enjoyment. These are the Bit Loader Program explained in
the Monitor Program.Doc file.

	Attached files:
	lmcmpldr.asm	This file is used to transfer the 0000h Monitor
Program into address 0000h
  	hmcmpldr.asm	This file is used to transfer the 8000h Monitor
Program into address 8000h

	Understanding that Tiny Basic was writing to never run out of a 1802
64K RAM system, I have changed that fact around so now the modified version
of Tiny Basic I have will run out of any 1802 64K RAM system. :-)

	Now that made it possible for my Monitor Program to be change to run
out of either ROM or RAM to accommodate Tiny Basic.

	Please note that both the 0000h and 8000h Monitor Programs are
identical except for where they got assembled to be loaded in at in a
system.
